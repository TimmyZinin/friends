## 1. Миссия

Создать открытый протокол, который соединяет людей по тому, **как они думают** — не по фотографиям, анкетам или социальному статусу. Протокол работает на стороне пользователя, шифрует данные до передачи, и позволяет каждому контролировать свою цифровую идентичность.

## 2. Проблема

Существующие решения для знакомств и нетворкинга:

| Проблема | Tinder, Bumble | LinkedIn | Discord/Slack |
|----------|--------------|----------|---------------|
| Матчинг по внешности | Да — фото = единственный фильтр | Нет | Нет |
| Централизованное хранение | Да — все данные на серверах Match Group | Да — Microsoft владеет всем | Да |
| Ручные анкеты | Да — люди врут | Да — резюме ≠ реальность | Нет матчинга |
| Матчинг по мышлению | Нет | Нет | Нет |
| Приватность | Нет — данные продаются | Нет — профиль публичен | Частичная |

**Ключевой инсайт из разговора:**
> *"Это возможность познакомиться с людьми которые тебе близки... друг о котором ты даже не знал что существует в мире"*

Люди хотят serendipity — случайных, но значимых встреч. Существующие платформы оптимизированы под engagement, а не под качество связей.

## 3. Решение: Friends Protocol

### Что это
Открытый протокол нетворкинга по знаниям. Не приложение, а **стандарт** — как HTTP для веба или ActivityPub для федеративных соцсетей.

### Как это работает (пользовательский путь)
```
1. Пользователь устанавливает скилл /friends в Claude Code (или другой MCP-клиент)
2. Скилл сканирует локальные файлы (markdown, экспорты чатов, заметки) с разрешения
3. На устройстве извлекаются темы (TF-IDF, NLP)
4. Темы кодируются в Bloom filter (необратимый битовый массив)
5. Bloom filter подписывается приватным ключом Ed25519
6. Только подписанный Bloom filter отправляется на matching engine
7. Engine вычисляет сходство (Jaccard) и возвращает Top-K совпадений
8. Пользователь видит интерактивный граф похожих "разумов"
9. Кликает на узел → видит, что человек решил показать → связывается через Telegram
```

### Ключевые принципы

1. **Privacy by architecture** — файлы никогда не покидают устройство
2. **Протокол, не приложение** — любой может построить клиент поверх Friends Protocol
3. **Открытый код, защищённый протокол** — BSL-лицензия, монетизация через protocol fees
4. **Блокчейн как следующий шаг** — от централизованного сервера к децентрализованной сети

## 4. Целевая аудитория

### Персона 1: AI-разработчик (MVP)
- **Кто:** Использует Claude Code ежедневно, имеет markdown-файлы
- **Боль:** Хочет найти единомышленников, не знает где искать
- **Путь:** Twitter → GitHub repo → устанавливает скилл → видит граф → находит интересных людей
- **Мотивация:** "Найти со-фаундера / ментора / друга с похожими интересами"
- **TAM:** ~500K+ пользователей Claude Code (растёт 10%/мес)

### Персона 2: Участник конференции (Phase 2)
- **Кто:** Приехал на tech-конференцию, не знает кого из 500 участников стоит найти
- **Боль:** Нетворкинг на конференциях хаотичен и неэффективен
- **Путь:** Организатор интегрирует Friends → участник устанавливает скилл → видит "кто из участников думает как я"
- **Мотивация:** "Максимизировать ценность конференции"
- **Revenue:** $2/участник (платит организатор)

### Персона 3: Casual user (Phase 3+)
- **Кто:** Не разработчик, но хочет найти интересных людей
- **Боль:** Устал от поверхностных связей в соцсетях
- **Путь:** Мобильное приложение (не CLI) → импорт данных из Telegram/Discord → граф
- **Мотивация:** "Найти людей, с которыми реально интересно общаться"

## 5. Источники данных (расширенные)

> *"Собирать всю информацию о человеке из всех его markdown файлов... опустосировать... закодировать на стороне человека"*

| Источник | Как загружается | Что извлекаем | Фаза |
|----------|----------------|---------------|------|
| Markdown файлы (*.md) | Скилл сканирует директории с разрешения | Темы, интересы, экспертиза | MVP |
| Claude session history | Из ~/.claude/ | Паттерны вопросов, темы диалогов | MVP |
| Telegram экспорт | Пользователь экспортирует чат → скармливает скиллу | Темы общения, интересы | Phase 2 |
| Discord экспорт | JSON export → скилл | Серверы, каналы, темы | Phase 2 |
| Slack экспорт | JSON export → скилл | Рабочие интересы, проекты | Phase 2 |
| Браузерные закладки | Export bookmarks → скилл | Темы исследований | Phase 2 |
| Notion export | Markdown export → скилл | Заметки, проекты, базы знаний | Phase 2 |
| Геолокация | Opt-in, city-level | Город (не GPS!) | MVP |
| Oura / Apple Health | API интеграция | Уровень стресса, активности | Phase 3 |
| Произвольные данные | Текстовый файл → скилл | Любые темы через NLP | Phase 2 |

**Принцип:** скилл принимает ЛЮБЫЕ текстовые данные. Unified pipeline: текст → NLP → topic tags → Bloom filter. Формат входа не важен — важен выход (теги тем).

## 6. Конкурентный ландшафт

| Продукт | Подход | Статус | Почему мы лучше |
|---------|--------|--------|----------------|
| **Tinder, Bumble** | Фото + свайпы | $3B+ revenue | Мы: мышление > внешность |
| **LinkedIn** | Резюме + connections | 1B+ users | Мы: реальные интересы > формальные навыки |
| **OkCupid** | Анкеты совместимости | Отказались от сложного матчинга | Мы: автоматическое извлечение, не ручные анкеты |
| **Pheramor** | ДНК-матчинг | Закрылся 2019 (Apple запретил) | Мы: не трогаем ДНК |
| **Lens Protocol** | Децентрализованный соцграф | <5K dating users | Мы: distribution через Claude Code skill (85K+ скиллов) |
| **Farcaster** | Децентрализованная соцсеть | Pivoted to wallets 2025 | Мы: фокус на matching, не на контент |
| **Polywork** | Нетворкинг для билдеров | $28M raised, pivoted to job search | Мы: protocol > app, privacy > profiles |

**Наша уникальность:**
1. **Единственный** matching по knowledge graph из AI-диалогов
2. **Единственный** протокол (не приложение) для people matching
3. **Единственная** distribution через AI skill ecosystem
4. **Единственный** с client-side zero-knowledge encoding

## 7. Фазы развития

### Phase 1: MVP (текущая, апрель-май 2026)
- Лендинг с интерактивным графом ✅
- GitHub repo (source of truth) ✅
- SDD документация ← в работе
- Claude Code скилл-прототип
- Centralized matching server (FastAPI + SQLite на Contabo)
- 30-50 alpha-пользователей

### Phase 2: Alpha (июнь-август 2026)
- Расширенные источники данных (Telegram, Discord, Slack exports)
- Feedback loop (thumbs up/down → weight adjustment)
- Event matching для конференций (первый revenue stream)
- Proximity matching (city-level)
- Multi-agent support (Cursor, Windsurf)
- 500+ пользователей

### Phase 3: Protocol & Scale (сентябрь-декабрь 2026)
- Open-source Friends Protocol спецификация
- Third-party клиенты
- Wearables интеграция (с legal review)
- Postgres + горизонтальное масштабирование
- 5,000+ пользователей

### Phase 4: Decentralization (2027+)
- Смарт-контракты на блокчейне (identity, consent, matching)
- Token economics (protocol fees, node rewards)
- Полная децентрализация matching engine
- 50,000+ пользователей

## 8. Позиционирование

### Одно предложение
**Friends Protocol** — открытый протокол, который соединяет людей по тому, как они думают, а не по тому, как они выглядят.

### Ключевые сообщения
1. *"Найди друга, о котором ты не знал"* — основной hook
2. *"Без фото. Без анкет. Без свайпов. Только разумы."* — anti-Tinder
3. *"Твои файлы остаются на твоём устройстве"* — приватность
4. *"Протокол, а не приложение"* — долгосрочное видение

### Категория
**Knowledge Networking Protocol** — новая категория, мы её создаём.

## 11. Что мы НЕ делаем (anti-scope)

Из разговора с Денисом были отклонены экспертной панелью:

| Идея | Почему отклонена |
|------|-----------------|
| ДНК-матчинг | GDPR Art.9, Pheramor закрылся, PR-катастрофа ("евгеника") |
| Автоматический расчёт дохода | Этически токсично, технически невозможно, GDPR Art.22 |
| Bluetooth mesh P2P | Слишком сложно, нет success stories (Briar <10K users) |
| Real-time GPS proximity | Инструмент для stalking |
| Физические параметры (рост, вес) | Core differentiator: minds > bodies |
| Блокчейн в MVP | 10x сложность, 0 value при <1K users |
